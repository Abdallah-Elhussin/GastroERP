## 1. ما هو GastroERP؟

**GastroERP** منصة ERP متكاملة للمطاعم والضيافة — ليست مجرد POS، بل نظام تشغيل كامل يجمع:

| المجال | الوظيفة |
|--------|---------|
| **POS & المبيعات** | طلبات، مدفوعات، ورديات، طاولات |
| **المطبخ** | KDS، محطات التحضير، تذاكر المطبخ |
| **المخزون** | مستودعات، وصفات، مشتريات، جرد |
| **المنيو** | منتجات، إضافات، أسعار متعددة |
| **الفوترة** | فواتير ZATCA، ضريبة، إشعارات دائن/مدين |
| **التوصيل** | مناطق، سائقين، تتبع |
| **CRM & الولاء** | عملاء، نقاط، كوبونات |
| **المحاسبة** | دليل حسابات، قيود، فترات مالية |
| **التقارير** | BI، KPIs، تصدير |
| **الأتمتة** | Jobs، إشعارات، تكاملات خارجية |
| **الذكاء الاصطناعي** | تنبؤ، توصيات، مساعد، كشف احتيال |
| **الموارد البشرية** | موظفين، حضور، إجازات، رواتب، توظيف |
| **سير العمل والموافقات** | Workflow مركزي، تفويض، تصعيد، سجل مراجعة |

**الهدف:** من مقهى واحد → سلسلة مطاعم enterprise متعددة الفروع والمستأجرين (Multi-Tenant).

## 2. الرؤية والرسالة

### الرؤية
بناء واحدة من أقوى منصات ERP للمطاعم في المنطقة — قادرة على المنافسة مع Foodics و Toast و Oracle Micros — مع جودة كود enterprise، أمان، وقابلية توسع.

### الرسمة
تبسيط عمليات المطعم عبر منصة واحدة: تقليل الهدر، رفع دقة المخزون، تحسين تجربة العميل، وتمكين القرارات من البيانات — **Online + Offline**.

### المبادئ الهندسية
- **DDD** — قواعد العمل في Domain
- **Clean Architecture** — Domain ← Application ← Infrastructure ← Presentation
- **CQRS + MediatR** — فصل القراءة عن الكتابة
- **Multi-Tenant** — عزل بيانات كل مستأجر
- **Advisory AI** — الذكاء الاصطناعي يقترح ولا ينفّذ عمليات مالية/مخزنية تلقائياً
- **Audit Trail** — تتبع كل عملية حساسة

## 3. هيكل المشروع (Backend)

```
Backend/src/
├── GastroErp.Domain/          ← الكيانات، Enums، Domain Events
├── GastroErp.Application/     ← Services، CQRS، Validators
├── GastroErp.Infrastructure/← Jobs، Email، Storage، AI Adapters
├── GastroErp.Persistence/     ← EF Core، Migrations، Configurations
└── GastroErp.Presentation/    ← API Controllers، Auth، Swagger
```

**Stack:** .NET 9 · EF Core · SQL Server · JWT · FluentValidation · MediatR

## 8. ملخص شامل — ما أُنجز حتى الآن

### ✅ Backend Core (Phases 1–22)
نظام ERP مطاعم **كامل المعمارية** يغطي:
- تشغيل POS يومي (طلب → مطبخ → دفع → وردية)
- مخزون ووصفات ومشتريات
- فوترة ZATCA-ready
- توصيل وCRM
- محاسبة مزدوجة القيد

### ✅ Reporting Layer (Phase 23)
40+ تقرير ولوحة KPIs — قراءة فقط من البيانات التشغيلية بدون تكرار.

### ✅ Automation Layer (Phase 24)
Jobs ليلية، إشعارات، Webhooks، Payment Gateway stubs — جاهز للربط الإنتاجي.

### ✅ AI Platform (Phase 25 — 5 مراحل فرعية)
من **بيانات موثوقة** → **تنبؤ** → **توصيات** → **تفاعل لغة طبيعية** → **ذكاء متقدم (احتيال، شرائح، churn)**.

### ✅ HR & Workforce (Phase 26)
موظفين، حضور، إجازات، رواتب، توظيف — مع ترحيل Payroll إلى Finance.

### ✅ Workflow & Approval Engine (Phase 27)
محرك موافقات مركزي — تعريفات، خطوات، شروط، تفويض، تصعيد، إشعارات، ومهام خلفية.

### ✅ Workflow Integration (Phase 28)
ربط HR، Purchasing، Inventory، POS بمحرك Workflow المركزي — بدون موافقات مكررة داخل الوحدات.

### ✅ Reporting & Analytics Platform (Phase 29)
منصة تقارير فوق Phase 23: لوحات معلومات قابلة للتخصيص، تعريفات تقارير، KPIs مع Snapshots، تصدير، جدولة، وPower BI stub.

### ⚠️ ما لم يُنجز بعد (Backend)
- تطبيق Migrations على DB إنتاج
- Frontend / Mobile / Kiosk UI
- Sync Agent production-hardening (Offline)
- Unit/Integration tests شاملة
- CI/CD pipeline كامل
- LLM/OpenAI production config (حالياً Heuristic)

- **Multi-Region** — نشر جغرافي
- **Real ML Models** — استبدال Heuristic بـ Azure ML / OpenAI
- **Marketplace** — تكاملات طرف ثالث (Uber Eats، HungerStation…)
- **HR & Payroll** — موظفين، رواتب، حضور
- **Franchise Management** — امتياز تجاري
